## Introduction 🚀

![Alt Text](/assets/projects/NASA/space.png)

As cislunar space grows increasingly crowded, spacecraft face heightened risks during navigation. Traditionally, space missions rely on Earth-based communication to determine their location. But what happens when communication is lost?

To solve this, NASA is developing **TRON** (Target & Range-adaptive Optical Navigation) — a next-gen system that allows spacecraft to determine their position independently, without Earth’s help.

TRON uses two methods:
1. **Planetary Triangulation** for long-range navigation by measuring angles between celestial bodies.
2. **Crater-Based Localization** for close-range navigation by detecting and matching craters seen in images with pre-mapped craters on the lunar surface.

This project focuses on **crater detection** — a key component of crater-based localization. Accurate crater recognition helps spacecraft understand their exact position, enabling autonomous operation in deep space.

---

## Project Objective 🌕

The goal is to build a system that automatically detects lunar craters in spacecraft images and fits an **ellipse** around each crater’s rim.

This is challenging due to:
- **Visual complexity**: Craters vary in size, lighting, and visibility. A single crater can appear drastically different depending on sun angle, terrain, and camera viewpoint.
- **Tough computing limits**: Our system must run on a CPU (no GPU), process high-res 5MP grayscale images in under 20 seconds, and use less than 4GB RAM — similar to the specs of a Raspberry Pi 5.

Together, these constraints demand **highly optimized deep learning models** that are fast, lightweight, and robust to noise and variability in real-world lunar imagery.


## Literature Review 🧠

Early crater detection used edge detection and hand-crafted features but struggled with lighting, shape irregularities, and noise.

Modern deep learning methods fall into two categories: single-stage (e.g., YOLO, SSD) for speed, and two-stage (e.g., Faster R-CNN, Mask R-CNN) for higher accuracy.

In this project, we compare YOLOv10 for real-time detection with Ellipse R-CNN, which fits elliptical crater shapes.

These models were selected for their balance of accuracy, efficiency, and relevance to the lunar navigation context.

## Dataset 📸

We used a synthetically generated lunar crater dataset developed by NASA, based on the crater catalog by Robbins (2019), rendered using the Boromir engine (Unreal Engine-based).

🗂️ **Dataset Composition**:  
- 1,000 grayscale images (2592×2048 px), each with a matching metadata file (e.g., sun angle, camera pose).
- Crater labels provided in both YOLO format (bounding boxes) and EllipseRCNN format (elliptical parameters).

## 4. Experiments

### Evaluation Metrics

We evaluate crater detection using Intersection over Union (IoU) and calculate Average Precision (AP) from precision-recall curves at multiple thresholds to assess accuracy and ranking quality.

![Alt Text](/assets/projects/NASA/evaluation_metrics.png)

### YOLOv10

We fine-tuned the YOLOv10m model on a custom, manually annotated crater detection dataset. 


YOLOv10m was selected as a middle-ground between accuracy and speed, ideal for real-time detection in planetary imagery.

Training involved 100 epochs using Ultralytics’ implementation, with adaptive learning rate (initially 0.01, settling around 0.002), dynamic batch sizing, and standardized image dimensions (640×640). The model used a composite loss function including Box Loss (MSE), Class Loss (cross-entropy), and Objectness Loss (binary cross-entropy).

Loss curves indicated smooth convergence and minimal overfitting. Inference on the validation set produced consistent bounding box predictions, with visual examples confirming accurate crater localization. While full metrics are still in development, the YOLOv10 pipeline sets a strong baseline for future comparison with Ellipse R-CNN.

After fine-tuning, our YOLOv10m model achieved 65% precision and 51% recall — so about two-thirds of predicted craters were correct, while it successfully detected roughly half of all actual craters. At an IoU threshold of 0.5, the model reached a mean Average Precision of 57%. 
Under the stricter COCO-style evaluation, which averages over IoUs from 0.5 to 0.95, the mAP was 34%. 
These results provide a strong initial baseline, especially for a first training run — with clear potential for improvement through further fine-tuning and architecture adaptation.

![Alt Text](/assets/projects/NASA/yolov10_initialresults.png)

### Ellipse R-CNN

EllipseRCNN, a specialized model for detecting elliptical objects, was fine-tuned on our crater dataset by freezing the backbone and updating the region proposal, ellipse regression, and occlusion handling modules. We initially applied inference on full-sized images (2592×2048), but observed many misdetections.

<p float="left">
  <img src="/assets/projects/NASA/ellipse_rcnn_wholeimage_initial.png" width="45%" />
  <img src="/assets/projects/NASA/ellipse_rcnn_wholeimage_results.png" width="45%" />
</p>

To improve performance, we implemented a tiling approach, dividing each image into 512×512 patches. This significantly enhanced the model’s ability to detect smaller craters and reduced false positives and negatives.

Training was conducted for 40 epochs using the Adam optimizer with a learning rate of 1e-4. The model converged around epoch 30. Evaluation on a 100-image hold-out set yielded a mean Average Precision (mAP) of 17.2% at IoU 0.5, and 9.9% at IoU 0.7. Further improvements are being explored through tile-based finetuning and uniform patch evaluation.

## 5. Next Steps

### 5.1 Evaluation Data Standardization
To ensure a fair comparison between EllipseRCNN and YOLOv10, we will define consistent training, validation, and testing splits. This unified dataset handling will allow us to compute evaluation metrics on the same images, enabling a more reliable model comparison.

![Alt Text](assets/projects/NASA/datasciencepipeline_midterm.png)

### 5.2 EllipseRCNN
We plan to incorporate tiling during training, not just inference, to help the model better detect smaller craters. Additionally, we will experiment with different finetuning strategies—adjusting optimizers, learning rate schedules, and training durations—to improve model performance.

### 5.3 YOLOv10
The YOLOv10 model will be trained on the full 1000-image dataset with a consistent split aligned with EllipseRCNN. We aim to apply tiling and augmentation to increase robustness, perform hyperparameter tuning, and extend training to 300 epochs. We also plan to experiment with larger YOLOv10 variants and apply unified evaluation metrics to directly compare results across models.
